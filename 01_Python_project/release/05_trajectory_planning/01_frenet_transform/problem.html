궤적 계획 — Frenet ↔ Cartesian 좌표 변환

Chapter 5 · Trajectory Planning / 01_frenet_transform  |  굽은 도로를 펴서 다루는 좌표계

굽은 도로 위의 자동차 위치를 전역 (x, y) 로 다루면 "차선 안에 있나", "앞차와 얼마나 떨어졌나" 같은 질문이 곧장 풀리지 않습니다. 도로 중심선을 따라간 거리 s 와 중심선에서 옆으로 벗어난 거리 d 로 위치를 기술하는 Frenet 좌표계와, 전역 (x, y) Cartesian 좌표계 사이를 오가는 두 변환을 구현해 보세요.

1문제 상황

굽은 3 차선 도로 위에 자차 1 대와 타겟 6 대가 놓여 있습니다. Frenet 좌표계는 도로 중심선을 기준 축으로 삼아, 위치를 두 값으로 분해합니다:

  • s — 종방향: 중심선을 따라간 호 길이(arc length). "도로를 따라 얼마나 멀리".
  • d — 횡방향: 중심선에서 옆으로 벗어난 부호 있는 거리. "어느 차선에, 차선 안 어디쯤". d > 0 = 진행 방향 기준 왼쪽.

이렇게 하면 굽은 도로가 곧게 펴진 것처럼 다뤄집니다 — 다음 과제 (02_frenet_moving_target)의 궤적 계획기는 이 펴진 좌표계 위에서 동작합니다. 이 과제는 그 좌표계 자체를 만드는 단계입니다.

2주어진 환경

아래 파일들은 검증 환경입니다 — 읽어서 인터페이스를 파악하되 수정하지 않습니다.

파일 역할
road.py 열린 곡선 3 차선 도로(LANE_WIDTH = 4 m, s ∈ [-50, +100]). Road 가 중심선 점열 center_x / center_y 와 누적 호 길이 center_s, 시각화용 차선 폴리라인을 제공합니다. 좌표 변환 자체는 포함하지 않습니다.
demo.py 시나리오(자차 + 타겟 6 대)를 구성하고, 변환 전후를 plotly 2-패널로 그립니다.

중심선은 점열 + 누적 호 길이로 주어진 폴리라인입니다. 이 도로는 s 의 원점(s = 0)이 자차의 종방향 위치라서, center_s 는 0 이 아니라 음수에서 시작합니다 — 주어진 center_s 를 그대로 사용하세요.

3구현 과제

frenet_transform.py 의 두 함수(# TODO)를 구현하세요. 시그니처는 이미 주어져 있습니다. 

def cartesian_to_frenet(x, y, cx, cy, cs) -> (s, d)
def frenet_to_cartesian(s, d, cx, cy, cs) -> (x, y, heading)
인자 의미
cx, cy 중심선 점들의 x / y 좌표 배열 (Road.center_x / center_y)
cs 중심선 각 점까지의 누적 호 길이 배열 (Road.center_s)

① cartesian_to_frenet — 전역 (x, y) → Frenet (s, d)

  • 점을 중심선의 각 세그먼트에 정사영해, 가장 가까운 발(foot)을 찾습니다. 세그먼트 보간 파라미터 t 는 [0, 1] 로 clamp 합니다.
  • s = 발까지의 누적 호 길이, d = 발에서 점까지의 수직 거리. d 의 부호는 진행 방향 기준 왼쪽(+) / 오른쪽(-) 입니다.

② frenet_to_cartesian — Frenet (s, d) → 전역 (x, y, heading)

  • s 가 속한 중심선 세그먼트를 찾아, 세그먼트 시작점에서 호 길이만큼 전진한 중심선 위 기준점을 구합니다.
  • 그 기준점에서 왼쪽 법선 방향으로 d 만큼 이동한 점이 결과 입니다. 도로 양 끝을 벗어난 s 는 clamp 합니다.

5합격 기준 pytest

알고리즘 형태는 자유 — 동작 명세(behavioral spec)만 검증합니다. 검증 명령은 4 절 참조.

  1. 중심선 복원 — d=0 인 Frenet 점은 중심선 위로 변환되고, 중심선 점은 (s, d≈0) 으로 되돌아온다.
  2. 횡 offset — frenet_to_cartesian(s, d) 가 중심선에서 왼쪽 법선 방향으로 거리 |d| 만큼 떨어진 점을 준다.
  3. 부호 규약 — 진행 방향 왼쪽으로 벗어난 점은 d>0, 오른쪽은 d<0 으로 변환된다.
  4. 변환 왕복 — cartesian → frenet → cartesian 왕복이 원래 점을 복원한다 (두 변환이 서로 역함수).

6힌트

  • 세그먼트 정사영 — 점 P 를 세그먼트 C_i → C_{i+1} 에 정사영하면 보간 파라미터 t = (P - C_i)·seg / |seg|². t 를 [0, 1] 로 clamp 하지 않으면 발이 세그먼트 밖으로 튀어 엉뚱한 세그먼트가 최단으로 뽑힙니다.
  • d 의 부호 — 거리 자체는 양수입니다. 세그먼트 방향과 "발→점" 벡터의 외적 z 성분 부호로 왼쪽(+)/오른쪽 (-)을 가릅니다.
  • 세그먼트 찾기 — frenet_to_cartesian 에서 s 가 속한 세그먼트는 numpy.searchsorted(cs, s) 로 찾을 수 있습니다. 경계 인덱스(첫/마지막 세그먼트) 처리에 주의하세요.
  • 두 변환의 정합 — cartesian_to_frenet 의 부호 판정과 frenet_to_cartesian 의 법선 방향이 같은 쪽(왼쪽 = +)을 가리켜야 왕복이 맞습니다.
  • s 원점은 자차 위치 — center_s 는 음수에서 시작합니다. 직접 0 부터 호 길이를 쌓지 말고 주어진 cs 를 그대로 쓰세요.
