궤적 계획 — Frenet 좌표계 최적 궤적 (이동 타겟 회피)

Chapter 5 · Trajectory Planning / 02_frenet_moving_target  |  샘플링 기반 최적 궤적 계획기의 핵심

여러분의 차량(ego)은 2 차선 폐루프 트랙을 주행합니다. 앞에는 차선을 유지하며 달리는 다른 차량(타겟)들이 있습니다. 매 시간 스텝마다 여러 후보 궤적을 만들어, 충돌 없이 가장 매끄럽고 목표 속도에 가까운 궤적을 고르는 Frenet 최적 궤적 계획기를 구현해 보세요.

1문제 상황

도로처럼 굽은 길에서는 전역 (x, y) 좌표로 궤적을 계획하기 까다롭습니다. 대신 도로 중심선을 따라간 거리 s 와 중심선에서 옆으로 벗어난 거리 d 로 위치를 기술하면(Frenet 좌표계), 굽은 도로에서도 주행을 "앞으로 얼마나(s)" + "어느 차선으로(d)" 의 두 1 차원 문제로 단순화할 수 있습니다.

매 스텝 계획기가 하는 일:

  • 후보 궤적 다발 생성 — 목표 차선(2 개) × 종방향 속도(3 개) × 종료 시간(4 개) 의 모든 조합으로 후보 궤적을 만듭니다.
  • 비용 평가 — 각 후보에 jerk(승차감) · 종료 시간 · 차선 일관성 · 목표 속도 오차로 비용을 매깁니다.
  • 최적 선택 — 속도·가속도·곡률 한계와 충돌 검사를 통과한 후보 중 비용이 최소인 궤적을 고릅니다.

ego 는 매 스텝 최적 궤적의 한 스텝 뒤 상태로 전진합니다(계획만 수행, 별도 제어기 없음). 느린 타겟을 만나면 계획기가 자연스럽게 차선을 바꿔 추월하게 됩니다.

2주어진 환경

아래 파일들은 검증 환경입니다 — 읽어서 인터페이스를 파악하되 수정하지 않습니다.

파일 역할
track_map.py 폐루프 2 차선 stadium 트랙(둘레 ≈ 200 m, LANE_WIDTH = 4 m) — 중심선 + Frenet 변환 + 시각화용 차선.
frenet_frame.py Frenet (s, d) ↔ 전역 (x, y) 좌표 변환 (폐루프 wrap 처리).
target_vehicles.py 타겟 차량 plant. Maneuver · TargetVehicle · TargetFleet. 타겟은 차선을 유지("straight")하며, 같은 차선에서 앞차에 근접하면 두 차가 속도를 맞바꿔(TargetFleet) 타겟끼리는 서로 추돌하지 않습니다.
frenet_planner.py 의 주어진 함수 calc_global_paths(Frenet→전역 변환), predict_target_cv(타겟 예측), collision_check / check_path(검사), frenet_optimal_planning(최적 선택) — 이미 구현되어 있습니다.

타겟 예측은 CV(등속) 모델입니다 — 계획기는 "타겟이 현재 속도로 차선을 유지한다"고만 가정합니다(predict_target_cv, 주어짐). 타겟의 실제 거동과 예측 모델은 분리되어 있으며, 더 정교한 예측은 다음 과제에서 다룹니다.

① 다항식 궤적 — QuinticPolynomial / QuarticPolynomial

Frenet 궤적은 시간에 대한 다항식으로 표현합니다. 양 끝의 운동 상태를 만족하도록 계수를 정합니다.

  • QuinticPolynomial (횡방향용, 5 차) — t=0 의 (위치, 속도, 가속도) 와 t=T 의 (위치, 속도, 가속도), 총 6 개 경계조건을 만족하는 6 개 계수.
  • QuarticPolynomial (종방향용, 4 차) — t=0 의 (위치, 속도, 가속도) 와 t=T 의 (속도, 가속도), 총 5 개 경계조건을 만족하는 5 개 계수. 종방향은 종료 위치를 구속하지 않습니다(velocity-keeping).

② 후보 궤적 생성 — calc_frenet_paths

def calc_frenet_paths(si, si_d, si_dd, sf_d, sf_dd,
                      di, di_d, di_dd, df_d, df_dd, opt_d) -> list[FrenetPath]
인자 의미
si, si_d, si_dd 종방향 시작 위치·속도·가속도
sf_d, sf_dd 종방향 종료 기준 속도·가속도 (SF_D_SET 이 sf_d 에 더해짐)
di, di_d, di_dd 횡방향 시작 위치·속도·가속도
df_d, df_dd 횡방향 종료 속도·가속도
opt_d 직전 스텝 최적 궤적의 종료 횡위치 — 차선 일관성 비용의 기준
반환 FrenetPath 후보 목록 — Frenet 시계열과 비용(c_lat, c_lon, c_tot)을 채움
  • SF_D_SET(종방향 속도) × DF_SET(목표 차선) × MIN_T..MAX_T(종료 시간) 의 모든 조합으로 후보를 만듭니다.
  • 종료 시간 T 가 MAX_T 보다 짧은 후보는 마지막 상태를 등속 외삽해 모든 궤적 길이를 MAX_T 로 통일합니다(비용 비교 공정성).
  • 비용 = jerk(3 차 미분의 제곱합) + 종료 시간 + 차선 일관성(opt_d 대비) + 목표 속도 오차. 가중치 상수는 파일 상단에 주어져 있습니다.

5합격 기준 pytest

알고리즘 형태는 자유 — 동작 명세(behavioral spec)만 검증합니다. 검증 명령은 4 절 참조.

  1. 5 차 다항식 경계조건 — QuinticPolynomial 이 t=0 / t=T 의 위치·속도·가속도 6 개 조건을 모두 만족.
  2. 4 차 다항식 경계조건 — QuarticPolynomial 이 t=0 의 위치·속도·가속도 + t=T 의 속도·가속도 5 개 조건을 만족.
  3. 후보 궤적 구조 — calc_frenet_paths 가 24 개(=속도 3 × 차선 2 × 종료시간 4) 후보를 반환, 각 Frenet 시계열 길이가 일치.
  4. 차선 후보 커버 — 후보들의 종료 횡위치가 두 차선 중앙을 모두 커버.
  5. 폐루프 주행 — 20 초 시뮬레이션 동안 타겟과 충돌 없이(최소 간격 충분) 트랙을 전진하고, 평균 속도가 목표 속도 근방.

6힌트

  • 다항식 계수 — a0, a1, a2 는 t=0 의 (위치, 속도, 가속도) 에서 바로 나옵니다(속도는 1 차 미분, 가속도는 2 차 미분 계수). 나머지 계수는 t=T 의 조건들이 만드는 작은 선형계(5 차→3×3, 4 차→2×2)를 numpy.linalg.solve 로 풀면 됩니다. 표준 식 유도는 강의자료를 참고하세요.
  • QuinticPolynomial 생성자에 가속도 항이 들어갈 때 a2 = ai/2 처럼 1/2 계수가 붙는 것에 주의하세요.
  • 후보 길이 통일 — 종료 시간 T 가 짧은 후보를 MAX_T 까지 등속 외삽하지 않으면, jerk 비용의 합이 길이에 따라 달라져 공정한 비교가 안 됩니다.
  • jerk 는 3 차 미분 — 비용의 jerk 항은 d_ddd / s_ddd (가속도가 아니라 그 미분)의 제곱합입니다.
  • 횡방향 궤적만 채운 FrenetPath 를 종방향용으로 복제할 때 copy.deepcopy 가 편리합니다.
  • 막히면 frenet_optimal_planning 등 주어진 함수가 calc_frenet_paths 의 반환값을 어떻게 쓰는지 읽어 보세요 — 채워야 할 필드가 분명해집니다.
