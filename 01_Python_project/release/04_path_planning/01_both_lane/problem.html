양쪽 차선 평균 — Both Lane Path Planning

Chapter 4 · Path Planning / 01_both_lane  |  차선 인식 결과로부터 주행 path 를 만드는 첫 단계

좌·우 차선이 모두 인식된 도로에서, 두 차선의 다항식 계수로부터 차로 중앙선 path 의 계수를 만들어 보세요. ego 차량은 이 path 를 Chapter 3 의 PurePursuit 컨트롤러로 추종합니다.

1문제 상황

자율주행의 lateral 파이프라인은 차선 인식 → path 생성 → 경로 추종 순서로 흐릅니다. Chapter 3 에서는 path(다항식 계수)가 이미 주어졌지만, 실제 차량은 카메라가 본 좌·우 차선을 먼저 path 로 바꿔야 합니다.

  • 차선 인식 모듈이 좌측 차선과 우측 차선을 각각 local frame 다항식 계수로 넘겨줍니다.
  • 여러분이 만들 것은 그 둘로부터 차로 중앙을 지나는 path 계수입니다.
  • 이번 과제는 양쪽 차선이 모두 valid 한 가장 단순한 경우 — sinusoidal 도로입니다.

2주어진 환경 — lateral 파이프라인

한 시간 스텝의 처리 흐름은 record_gen 이 모두 엮어 줍니다. 여러분이 채우는 것은 그 중 path 생성 한 칸뿐입니다. 

차선 계수 (coeff_L, coeff_R)        ← lane_both.py 도로 + 차선 인식
    │
    ▼  both_lane_to_path()           ← 이번 과제 (구현 대상)
중앙선 path 계수
    │
    ▼  PurePursuit.step()            ← Chapter 3 결과물 (재사용)
조향각 δ
    │
    ▼  VehicleLat.step()             ← vehicle_lat_both.py 차량 모델
  • lane_both.py — sinusoidal 도로와 좌·우 차선을 생성하는 시나리오 환경.
  • vehicle_lat_both.py — ego 의 lateral 차량 모델.
  • PurePursuit — Chapter 3 07_pure_pursuit 의 컨트롤러를 sys.path 로 import 해 재사용.

3구현 과제

both_lane_planner.py 의 both_lane_to_path() 함수 본문(# TODO)을 구현하세요. 시그니처는 이미 주어져 있습니다. 

def both_lane_to_path(coeff_L: np.ndarray, coeff_R: np.ndarray) -> np.ndarray
    """좌·우 차선 계수로부터 중앙선 path 계수를 반환."""
인자 의미
coeff_L 좌측 차선 다항식 계수 — shape (degree+1, 1) column, 고차→저차 순
coeff_R 우측 차선 다항식 계수 — 같은 shape
반환 중앙선 path 계수 — 입력과 동일 shape (degree+1, 1) column
  • coeff_L / coeff_R 은 Chapter 3 의 PolynomialFitting.fit() 출력 형태로, local frame 의 좌/우 차선 다항식입니다.
  • numpy 배열 산술은 element-wise 라서 계수 결합을 직관적으로 작성할 수 있습니다.

게인·파라미터 튜닝

이 과제에서 구현하는 것은 알고리즘 로직 입니다 — 라이브러리 코드의 시그니처는 고정이고 본문(# TODO)만 채웁니다. 게인·파라미터의 실제 값 은 코드가 아니라 실행 파일에서 정하며, 세 군데의 역할이 다릅니다.

  • 라이브러리 코드 (구현 대상) — 알고리즘만 구현. 게인·파라미터는 매개변수로 받기만 하고 값은 담지 않음.
  • producer (record_gen.py) — 게인·파라미터 값을 정하는 곳. release 기본값은 0 이라 그대로 실행하면 응답이 없거나 오류가 납니다 → 직접 값을 채워 실행하며 응답 변화를 비교하세요 (자유롭게 변경 OK).
  • test_*.py — 합격 기준 검증용 값. 변경 금지.

이 문제에서 바꾸는 줄 — record_gen.py 의 아래 줄. release 기본값은 0 이니 여기에 값을 채우세요:

    pp = PurePursuit(L=4.0, lookahead_time=0.0)

4실행 방법

모든 명령은 git 저장소 루트에서 실행합니다.

구현 파일

01_Python_project/release/04_path_planning/01_both_lane/both_lane_planner.py

테스트 — 합격 기준 검증

uv run pytest 01_Python_project/release/04_path_planning/01_both_lane/ -v

시나리오 실행 → record JSON 생성 + Rerun viewer 자동 실행

uv run python 01_Python_project/release/04_path_planning/01_both_lane/record_gen.py

파란 차량이 sinusoidal 도로의 차로 중앙선을 따라 주행합니다 — 양쪽 흰색 차선 + dashed 중앙선 + 매 step 의 planner path(노란 곡선)가 ego 앞으로 뻗습니다. lookahead_time 은 release 기본값이 0 이라 그대로 실행하면 추종이 되지 않습니다 — record_gen.py 안에서 직접 값을 채워 응답 변화를 비교하세요. JSON 만 만들고 viewer 를 띄우지 않으려면 --no-viewer 를 붙입니다.

Rerun viewer 재생

uv run python 01_Python_project/release/04_path_planning/simulator_path_planning.py 01_Python_project/release/04_path_planning/01_both_lane/

simulator_path_planning.py 를 인자 없이 실행하면 04_path_planning/ 하위의 vehicle 시나리오(01~03·06)를 한 viewer 에 멀티 로드해 좌측 Recordings 패널에서 클릭 전환할 수 있습니다.

디버그 신호 — 심화 분석 (선택)

record_gen.py 의 dbg.add(...) 블록에는 debug1/2/3 주석 placeholder 가 준비돼 있습니다. 주석을 풀고 분석하고 싶은 값·식(예: 추적 오차, 내부 상태)을 채우면 그 신호가 record 에 함께 저장됩니다 — 신호 추가·삭제·수정은 이 dbg.add() 호출의 kwarg 한 줄로 끝납니다.

디버그 신호는 viewer 기본 화면에는 나타나지 않습니다 — Rerun viewer 좌측 entity 패널에서 /debug/<이름> 항목을 골라 TimeSeriesView 를 직접 추가하면 시계열로 볼 수 있습니다. 구현이 막힐 때 내부 신호를 들여다보는 디버깅 도구로 활용하세요.

5합격 기준 pytest

알고리즘 형태(정통 좌·우 평균 / 다른 결합 방식)는 자유 — 동작 명세(behavioral spec)만 검증합니다.

  1. 인터페이스 sanity — both_lane_to_path 반환 shape 이 입력과 동일한 column.
  2. 폐루프 추적 오차 — sinusoidal both-lane 도로 30초 추종, tail 평균 |lateral err| < 0.3 m, peak < 1.0 m.
주의: trivial 구현(coeff_path = 0)은 도로 곡선을 따라가지 못해 임계값 초과로 차단됩니다.

6힌트

  • 중앙선은 좌·우 차선의 가운데 입니다 — 같은 차수의 계수끼리 어떻게 결합하면 가운데가 될지 생각해 보세요.
  • 다항식 계수는 column 벡터입니다. numpy 산술은 element-wise 라 차수별 계수에 한 번에 적용됩니다.
  • 반환 shape 은 입력과 같아야 합니다 — 새 배열의 형태를 먼저 확인하세요.
