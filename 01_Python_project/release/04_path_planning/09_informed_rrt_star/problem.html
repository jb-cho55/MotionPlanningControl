Informed RRT* — 타원 informed 경로 탐색

Chapter 4 · Path Planning / 09_informed_rrt_star  |  찾은 경로로 탐색 공간을 좁혀 정밀하게 수렴

Informed RRT* 로 격자 미로의 경로를 단계적으로 줄이세요. 한 번 경로를 찾으면, 그보다 짧은 경로가 있을 수 있는 영역은 start·goal 을 초점으로 하는 타원 안뿐입니다 — 그 타원 안에서만 다시 탐색하며, 더 짧은 경로를 찾을 때마다 타원이 좁아져 탐색이 점점 정밀해집니다.
이 과제는 차량 제어가 없습니다. 결과는 Rerun 2D viewer 의 tree 성장·수렴과 최종 경로로만 확인합니다.

1문제 상황

RRT*(08)는 tree 를 한 번 키워 경로를 냅니다 — 전체 공간을 고르게 sampling 하므로, 답이 절대 지날 리 없는 먼 구석까지 점을 뿌립니다. Informed RRT* 는 한 번 경로를 찾은 뒤 낭비를 없앱니다: 비용 c 이하의 경로는 반드시 start·goal 을 초점, c 를 장축 으로 하는 타원 안을 지납니다. 그러니 그 타원만 다시 sampling 하면 됩니다. 경로가 짧아질수록 타원이 작아지고, 탐색이 점점 정밀해집니다.

2Map 구조 S-커브 슬라럼

  • 60×60 격자, 외곽 border.
  • start→goal 일직선을 막는 세로 벽 2개 + 모서리 장식 사각형 4개 — 손으로 배치한 고정 맵이라 항상 같습니다.
  • 두 벽의 통과 구멍(gap)이 위·아래로 엇갈려 있어, 경로는 1번 벽을 위로 2번 벽을 아래로 감아 도는 완만한 S-커브가 됩니다.
  • start (10, 30), goal (50, 30) — 같은 높이지만 직선 경로는 슬라럼 벽이 차단합니다.

    goal: tuple[float, float],

) -> list[tuple[float, float]]

디버그 신호 — iteration 별 분석 (선택)

informed_rrt_star() 는 optional 인자 dbg 를 받습니다. 탐색 루프 안에서 dbg 가 주어졌을 때 dbg.add(...) 를 호출하면, 매 iteration 의 내부 값이 record 의 debug_scalars 로 저장됩니다 — goal 거리·reject·tree 크기·재배선 수· 현재 round·현재 최단 비용(best_cost) 같은 값을 남깁니다. 

for it in range(max_iter):
    ...
    if dbg is not None:
        dbg.add(goal_dist=..., rejected=..., tree_size=...,
                rewire_count=..., inform_round=..., best_cost=...)   # 매 iteration 한 줄

iteration 은 round 가 바뀌어도 이어지는 전역 카운터이므로, 디버그 신호도 모든 round 가 하나의 연속 타임라인으로 보입니다 — best_cost 가 round 마다 계단식으로 내려가는 것이 informed 수렴입니다. 좌측 entity 패널에서 /debug/<이름> 을 골라 TimeSeriesView 를 추가하세요.

6합격 기준 pytest

알고리즘 형태(자료 구조·RNG 구현)는 자유 — 동작 명세(behavioral spec)만 검증합니다.

  1. start 일치 — path[0] == start.
  2. goal 근접 — ‖path[-1] − goal‖ < goal_range(기본 8.0).
  3. segment 충돌 없음 — 연속 두 점을 잇는 선분의 fine-sample 어느 cell 도 obstacle 가 아님.
  4. segment 제약 — 연속 segment 길이 ≤ search_radius (choose-parent·rewire 연결 반경).
  5. on_step 콜백 호출 — 호출 수 ≥ path edge 수, 첫 호출 parent == start, iteration 단조 증가.
  6. 재현성 — 같은 seed 면 같은 path.
  7. budget 단조성 — max_iter 가 클수록 goal 도달 추정비용(경로 길이 + 마지막 node→goal 거리)이 작거나 같음 — 더 많은 iteration 이 경로를 악화시키지 않음.

7힌트

  • 단일 tree·단일 loop — tree 자료구조(nodes·parents·children·cost)는 loop 밖에서 한 번만 만듭니다. for it in range(max_iter) 안에서 08 RRT* 의 inner loop(sample / nearest / steer / collision-free / near / choose-parent / rewire)를 그대로 수행하며 같은 tree 를 계속 키웁니다. inform_round 는 별도 loop 가 아니라 "몇 번째 개선인가"를 세는 카운터입니다.
  • 샘플링 공간 — 첫 경로를 찾기 전(c_best 가 아직 무한)은 전체 [0, grid_size]² uniform. 첫 경로 후에는 현재 최단 경로 길이 c_best 로 _sample_in_ellipse(rng, start, goal, c_best) 를 호출해 타원 안에서만 샘플합니다. (goal bias 는 공통.)
  • 개선 검사 → round 진행 — 매 iteration goal_range 안 node 중 cost + goal 까지 직선거리 가 최소인 경로를 복원하고, 그 goal 도달 추정비용 eff 가 c_best − improve_eps 보다 작으면 — 의미 있게 더 짧으면 — c_best·최단 경로를 갱신하고 inform_round 를 1 올립니다. 그 순간 타원이 좁아집니다.
  • round schedule — eta 는 eta · eta_decay^inform_round, goal_range 는 goal_range · goal_decay^inform_round 로 작아집니다 (각각 eta_min / goal_range_min 으로 하한 clip). 첫 경로 전 inform_round=0 은 원래 값입니다.
  • iteration 은 단일 loop 카운터 — it 를 그대로 on_step 에 넘기면 on_step·디버그 타임라인이 자연히 이어집니다.
  • 반환은 max_iter 까지의 최단 경로. 끝내 경로를 못 찾으면 빈 list 를 반환합니다.
