Dijkstra — 8-connected 격자 최단 경로

Chapter 4 · Path Planning / 04_dijkstra  |  graph search 의 출발점 — 비용 누적 기반 탐색

60×60 격자 미로에서 start 부터 goal 까지의 8-connected 최단 경로를 Dijkstra 알고리즘으로 탐색하세요. 단계별 노드 expansion 과정을 Rerun 2D viewer 로 재생합니다.
이 과제는 차량 제어가 없습니다. 격자 탐색 알고리즘 자체에만 집중하세요 — 결과는 2D viewer 의 노드 expansion 과 최종 경로로만 확인합니다.

1문제 상황

01~03 은 path 가 연속 곡선(다항식)이었습니다. 이번 챕터 후반(04~07)은 격자/이산 공간의 graph search 로, path 가 노드의 나열입니다. Dijkstra 는 그 중 가장 기본 — 시작점부터의 누적 비용이 가장 작은 노드를 매 스텝 확장합니다.

  • start (5, 5), goal (55, 5) — 둘 다 벽 아래쪽이라 직진 경로가 없습니다.
  • 중앙 벽 위쪽(y ≥ 51)으로 우회해야 goal 에 닿습니다.

2Map 구조

  • 외곽 border(x=0,60 / y=0,60) — 완전 폐쇄.
  • 중앙 vertical wall(x=30, y=0..50) — 좌·우 분리. y=51..59 만 개방.
  • y=20 horizontal wall — x=0..20 ∪ x=40..60 폐쇄(x=21..39 개방).
  • y=35 horizontal wall — x=10..50 폐쇄(x=0..9 ∪ x=51..59 개방).

8-connected 이동

직진 이동 비용 1, 대각선 이동 비용 √2. 8방향 이동은 dijkstra.py 의 _ACTIONS 상수에 이미 정의돼 있습니다.

3구현 과제

dijkstra.py 의 dijkstra() 함수 본문(# TODO)을 구현하세요. 시그니처는 이미 주어져 있습니다. 

def dijkstra(
    start: tuple[int, int],
    goal: tuple[int, int],
    obstacles: set[tuple[int, int]],
    on_step: Callable[[tuple[int, int], set[tuple[int, int]], float, float], None] | None = None,
) -> list[tuple[int, int]]
인자 의미
obstacles 벽 셀의 집합 — set 이라 포함 검사가 O(1).
on_step 매 노드 expand 직후 호출되는 시각화 콜백 — (current_node, open_set, g_cost, f_cost). open_set 은 그 시점의 open list, g_cost 는 current 의 누적 비용, Dijkstra 는 heuristic 이 없어 f_cost = g_cost. None 이면 호출 안 함.
반환 경로 list(start → goal 순서). 경로 미발견 시 [].

게인·파라미터 튜닝

이 과제는 알고리즘 로직만 구현합니다 — 튜닝할 게인·파라미터가 없습니다. record_gen.py 는 시각화 viewer 를 띄우는 역할만 하며(--skip 등 viz 옵션만 존재), 탐색 동작에 영향을 주는 값은 없습니다. 합격 기준 값은 test_*.py 에 있으며 변경 금지 입니다.

2D viewer 에 격자 + 장애물(검정) + start(파랑) + goal(빨강)이 표시됩니다. 좌측 timeline scrubber step 을 움직이면 매 step 의 current(주황), 누적 closed list(옅은 노랑), 현재 open list(초록)가 갱신되고, 마지막 step 뒤 최종 path(마젠타)가 그려집니다.

  • JSON 만 만들고 viewer 를 띄우지 않으려면 --no-viewer.
  • frame 이 너무 많아 scrubber 가 답답하면 --skip N 으로 N step 마다 frame 1개로 묶습니다 (기본 10, --skip 1 = subsample 없음).

디버그 신호 — expansion 별 분석 (선택)

record_gen.py 의 on_step 콜백은 매 노드 확장(expansion)마다 호출됩니다 — 거기서 goal_dist(현재 노드→goal 거리)·open_size (open list 크기)·g_cost(시작→current 누적 비용)·h_cost(heuristic 항 — Dijkstra 는 항상 0)·f_cost(총 비용 g + h)를 record 의 debug_scalars 로 저장합니다. 더 분석할 값은 dbg.add(...) 의 kwarg 한 줄로 추가할 수 있습니다.

디버그 신호는 viewer 기본 화면에는 없습니다 — 좌측 entity 패널에서 /debug/<이름> 을 골라 TimeSeriesView 를 추가하세요. 하단 타임라인은 기본이 iteration (expansion 단위)이라 디버그 곡선이 expansion 축으로 곧바로 보입니다. frame 단위 (--skip 묶음)로 재생하려면 타임라인 드롭다운에서 step 을 고르세요.

5합격 기준 pytest

알고리즘 형태(정통 Dijkstra / heapq priority queue / 다른 변종)는 자유 — 동작 명세(behavioral spec)만 검증합니다.

  1. 시작·끝 일치 — path[0] == start, path[-1] == goal.
  2. 장애물 회피 — 모든 path 노드가 obstacles 밖.
  3. 8-connectivity — 연속 노드 간 Δx, Δy ∈ {−1, 0, 1}(이동량 0 아님).
  4. 최단 cost — path 총 cost(Σ √(Δx² + Δy²))가 reference 의 1.05배 이내.
  5. on_step 콜백 호출 — 콜백 전달 시 첫 호출의 current 가 start, 호출 횟수 ≥ path 길이.
주의: trivial 구현(BFS 로 step 수만 최소화)은 8-connected 의 대각 비용이 √2 라서 4번 최단 cost 에서 차단될 수 있습니다.

6힌트

  • 자료 구조 — 후보 노드(open)와 이미 확장한 노드(closed)를 따로 둡니다. 각 노드에 대해 시작점부터의 누적 cost 와 부모 노드를 함께 기억하면, goal 도달 후 부모를 거꾸로 따라가며 경로를 복원할 수 있습니다.
  • 매 스텝 — open 중 누적 cost 가 가장 작은 노드를 골라 closed 로 옮기고, 그 직후 on_step(current, open_set, g_cost, f_cost) 를 호출합니다(콜백이 있을 때 — Dijkstra 는 f = g).
  • 이웃 확장 — _ACTIONS 8방향에 대해 자식 후보를 만들고 obstacles·closed 를 검사합니다. 이미 더 작은 cost 로 도달한 노드는 갱신하지 않습니다.
  • open 이 비었는데 goal 에 닿지 못했다면 경로가 없는 것 — 빈 list 를 반환합니다.
  • 경로 복원 후에는 start → goal 순서가 되도록 뒤집어 반환하세요.
