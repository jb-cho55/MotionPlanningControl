RRT* — 최적 재배선 Rapidly-exploring Random Tree

Chapter 4 · Path Planning / 08_rrt_star  |  sampling tree 를 점점 더 짧은 경로로 키운다

RRT* 로 격자 미로에서 start 부터 goal 까지 경로를 탐색하세요. RRT(07)와 같은 map·같은 seed 를 쓰되, 새 노드를 가장 싸게 잇고(choose-parent) 주변 노드를 다시 잇는(rewire) 두 연산을 더해 — 같은 sampling 순서인데도 더 짧은 경로를 얻습니다.
이 과제는 차량 제어가 없습니다. 결과는 Rerun 2D viewer 의 tree 성장·재배선과 최종 경로로만 확인합니다.

1문제 상황

RRT(07)는 새 노드를 가장 가까운 tree 노드에 무조건 붙입니다 — 빠르지만, 먼저 닿은 경로가 그대로 답이라 멀리 우회한 가지여도 그냥 씁니다. RRT* 는 두 가지를 더합니다: ① 새 노드를 반경 안 노드들 중 start 부터의 누적 비용이 가장 작아지는 부모에 잇고(choose-parent), ② 추가한 뒤 주변 노드들을 새 노드 경유가 더 싸면 부모를 갈아끼웁니다(rewire). tree 가 자랄수록 경로가 최적에 가까워집니다.

디버그 신호 — iteration 별 분석 (선택)

rrt_star() 는 optional 인자 dbg 를 받습니다. 여러분이 구현한 탐색 루프 안에서 dbg 가 주어졌을 때 dbg.add(...) 를 호출하면, 매 iteration(reject 포함)의 내부 값이 record 의 debug_scalars 로 저장됩니다 — goal 까지 거리·reject 여부·tree 크기· near 후보 수·재배선 수 같은 값을 남겨 탐색이 어떻게 수렴하는지 분석할 수 있습니다. 

for it in range(max_iter):
    ...
    if dbg is not None:
        dbg.add(goal_dist=..., rejected=..., tree_size=...,
                near_count=..., rewire_count=...)   # 매 iteration 한 줄

Rerun viewer 는 search record 를 열 때 iteration 타임라인을 기본으로 잡습니다 — 하단 타임라인을 스크럽하면 tree 가 자라고 재배선되는 모습이 iteration 단위로 보입니다. 디버그 신호(scalar)는 기본 화면에 없으니, 좌측 entity 패널에서 /debug/<이름> 을 골라 TimeSeriesView 를 추가하면 같은 iteration 축에서 tree 와 함께 분석할 수 있습니다.

6합격 기준 pytest

알고리즘 형태(자료 구조·RNG 구현)는 자유 — 동작 명세(behavioral spec)만 검증합니다.

  1. start 일치 — path[0] == start.
  2. goal 근접 — ‖path[-1] − goal‖ < goal_range(기본 1.5).
  3. segment 충돌 없음 — 연속 두 점을 잇는 선분의 fine-sample 어느 cell 도 obstacle 가 아님.
  4. segment 제약 — 연속 segment 길이 ≤ search_radius (choose-parent·rewire 의 연결 반경).
  5. on_step 콜백 호출 — 호출 수 ≥ path edge 수, 첫 호출 parent == start.
  6. 재현성 — 같은 seed 면 같은 path.

7힌트

  • RRT 위에 3가지를 얹습니다 — sample / nearest / steer / collision-free / add 의 5단계는 07 RRT 와 같습니다. RRT* 는 add 자리에 near → choose-parent → rewire 가 들어갑니다.
  • cost 를 함께 관리 — node 마다 start 부터의 누적 거리 cost 를 dict 로 기억합니다. cost[start] = 0, 새 node 의 cost = 부모 cost + 부모까지 거리.
  • near — 새 점에서 search_radius 안에 있는 모든 tree node 의 index 를 모읍니다.
  • choose-parent — near 중 _is_collision_free 로 이을 수 있고 cost[i] + ‖i − new‖ 가 가장 작은 node 를 부모로. 후보가 없으면 nearest (reject 단계를 통과했으니 충돌 없음 보장).
  • rewire — 새 node 추가 후 near 의 각 node 에 대해, cost[new] + ‖new − i‖ < cost[i] 이고 충돌이 없으면 그 node 의 부모를 new 로 교체합니다. 부모가 바뀐 node 의 자손 cost 도 함께 갱신해야 이후 choose-parent 가 정확합니다 (자식 index 를 dict 로 기록해 두면 subtree 갱신이 쉽다).
  • RNG 는 인스턴스로 — random.Random(seed). on_step 은 node 추가와 rewire 양쪽에서 호출하세요. max_iter 를 다 돌고도 goal 에 닿지 못하면 빈 list 를 반환.
