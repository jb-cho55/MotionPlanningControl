가상 차선 — Virtual Lane Path Planning

Chapter 4 · Path Planning / 02_virtual_lane  |  한쪽 차선이 끊기는 구간을 처리하는 path 생성

실제 차선 인식은 항상 양쪽이 둘 다 보이지 않습니다 — 한쪽이 끊기는 구간이 흔하죠. valid 한 쪽 차선과 기억해 둔 차선폭으로 가상 차선을 세워, path 가 끊기지 않게 만들어 보세요.

1문제 상황

01_both_lane 은 양쪽 차선이 모두 valid 한 이상적인 경우였습니다. 이번에는 차선이 한쪽씩 끊기는 도로입니다.

  • X ∈ [20, 40] 구간 — 왼쪽 차선 invalid.
  • X ∈ [60, 80] 구간 — 오른쪽 차선 invalid.
  • 한쪽만 보일 때 path 가 끊기면 ego 가 차로를 이탈합니다.

핵심 아이디어: 양쪽이 모두 valid 한 구간에서 차선폭을 측정해 기억해 두고, 한쪽만 valid 인 시점에는 그 기억된 폭으로 반대쪽 가상 차선을 세웁니다.

2주어진 환경

record_gen 이 매 스텝 좌·우 차선 계수와 각 차선의 valid 여부를 함께 넘겨줍니다. 여러분이 채우는 것은 차선폭 추정과 가상 차선 path 생성입니다.

  • lane_virtual.py — 차선이 한쪽씩 끊기는 시나리오 도로.
  • vehicle_lat_virtual.py — ego 의 lateral 차량 모델.
  • PurePursuit — Chapter 3 의 컨트롤러를 import 해 path 추종에 재사용.

virtual_lane_planner.py 의 클래스·함수 본문(# TODO)을 구현하세요. 시그니처는 이미 주어져 있습니다.

요소 역할
LaneWidthEstimator.update 양쪽 valid 일 때만 차선폭 self.Lw 를 갱신 — 그 외에는 직전 값을 유지(latch).
either_lane_to_path valid 조합 4가지에 따라 path 계수를 반환 — 입력과 동일 column shape.

게인·파라미터 튜닝

이 과제에서 구현하는 것은 알고리즘 로직 입니다 — 라이브러리 코드의 시그니처는 고정이고 본문(# TODO)만 채웁니다. 게인·파라미터의 실제 값 은 코드가 아니라 실행 파일에서 정하며, 세 군데의 역할이 다릅니다.

  • 라이브러리 코드 (구현 대상) — 알고리즘만 구현. 게인·파라미터는 매개변수로 받기만 하고 값은 담지 않음.
  • producer (record_gen.py) — 게인·파라미터 값을 정하는 곳. release 기본값은 0 이라 그대로 실행하면 응답이 없거나 오류가 납니다 → 직접 값을 채워 실행하며 응답 변화를 비교하세요 (자유롭게 변경 OK).
  • test_*.py — 합격 기준 검증용 값. 변경 금지.

이 문제에서 바꾸는 줄 — record_gen.py 의 아래 줄. release 기본값은 0 이니 여기에 값을 채우세요:

    pp = PurePursuit(L=4.0, lookahead_time=0.0)

디버그 신호 — 심화 분석 (선택)

record_gen.py 의 dbg.add(...) 블록에는 debug1/2/3 주석 placeholder 가 준비돼 있습니다. 주석을 풀고 분석하고 싶은 값·식(예: 추적 오차, 내부 상태)을 채우면 그 신호가 record 에 함께 저장됩니다 — 신호 추가·삭제·수정은 이 dbg.add() 호출의 kwarg 한 줄로 끝납니다.

디버그 신호는 viewer 기본 화면에는 나타나지 않습니다 — Rerun viewer 좌측 entity 패널에서 /debug/<이름> 항목을 골라 TimeSeriesView 를 직접 추가하면 시계열로 볼 수 있습니다. 구현이 막힐 때 내부 신호를 들여다보는 디버깅 도구로 활용하세요.

5합격 기준 pytest

알고리즘 형태(가상 차선 보정·차선폭 추정 디테일)는 자유 — 동작 명세(behavioral spec)만 검증합니다.

  1. 인터페이스 sanity — either_lane_to_path 반환 shape 이 입력과 동일한 column.
  2. latch 동작 — LaneWidthEstimator 가 양쪽 valid 일 때만 갱신하고, 한쪽만 valid 면 직전 값을 유지.
  3. invalid 구간 폐루프 — 30초 시뮬, tail 평균 |lateral err| < 0.4 m, peak < 1.5 m.
주의: trivial 구현(차선폭 무시, 한쪽만 valid 시 zero coeff 등)은 invalid 구간에서 큰 peak 으로 차단됩니다.

6힌트

  • 차선폭 추정 — 양쪽 차선이 모두 valid 할 때, 좌·우 다항식의 상수항(local frame 의 y 절편) 차이가 곧 차선폭입니다. 그 조건일 때만 self.Lw 를 갱신하세요.
  • 가상 차선 — 한쪽만 valid 하면 valid 한 쪽 계수를 복사한 뒤 상수항을 차선폭의 절반만큼 반대 방향으로 옮기면 끊긴 차선을 세울 수 있습니다. 좌·우 방향에 따라 부호가 반대입니다.
  • valid 조합 4가지 — 양쪽 valid / 왼쪽만 / 오른쪽만 / 둘 다 invalid. 둘 다 invalid 면 직진(zero coeff)이 안전한 fallback 입니다.
  • 계수를 in-place 로 바꾸지 않도록 .copy() 후 수정하세요 — 원본 차선 계수가 오염될 수 있습니다.

7제약

  • lane_virtual.py, vehicle_lat_virtual.py, record_gen.py, test_virtual_lane.py — 수정 금지 (검증 환경).
  • LaneWidthEstimator.__init__(Lw_init=4.0) 의 기본값 변경 금지 — 테스트 fixture.
  • 클래스·함수의 시그니처 변경 금지.
