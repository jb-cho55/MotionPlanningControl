이동 타겟 추종 — Following Moving Target

Chapter 4 · Path Planning / 03_following_moving_target  |  앞 차량의 자취를 path 로 만들어 추종

앞 차량(leading)이 차로 중앙을 따라가는 동안, 뒤따르는 ego 가 leading 의 위치 history 를 누적해 그 자취를 cubic path 로 추종해 보세요.

1문제 상황

01·02 는 차선이 path 의 출처였습니다. 이번에는 차선 대신 앞 차량이 path 의 출처입니다 — leading 이 지나간 자리를 따라 ego 가 주행합니다.

핵심 아이디어: leading 의 위치를 매 스텝 ego 의 local frame 으로 측정해 history 로 쌓습니다. ego 자신이 움직일 때마다 그 history 점들을 새 ego frame 으로 갱신(역회전 + 역시프트)하고, 충분히 쌓이면 그 점들을 지나는 3차 다항식으로 path 를 만듭니다.

2주어진 환경

leading 차량은 record_gen 이 Chapter 3 의 lane keep 으로 운전합니다. ego 는 여러분이 만든 path 를 PurePursuit 으로 추종합니다.

  • lane_following.py — leading 이 달리는 sinusoidal 차로 시나리오.
  • vehicle_lat_following.py — ego·leading 의 lateral 차량 모델.
  • PurePursuit — Chapter 3 컨트롤러. leading 운전용·ego 추종용으로 각각 쓰입니다.

LeadingTargetTracker.update 새 측정을 history 에 추가하고, ego 가 한 스텝 움직인 만큼 기존 점들을 새 frame 으로 변환. 윈도우 길이 초과 시 가장 오래된 항목 제거.

게인·파라미터 튜닝

이 과제에서 구현하는 것은 알고리즘 로직 입니다 — 라이브러리 코드의 시그니처는 고정이고 본문(# TODO)만 채웁니다. 게인·파라미터의 실제 값 은 코드가 아니라 실행 파일에서 정하며, 세 군데의 역할이 다릅니다.

  • 라이브러리 코드 (구현 대상) — 알고리즘만 구현. 게인·파라미터는 매개변수로 받기만 하고 값은 담지 않음.
  • producer (record_gen.py) — 게인·파라미터 값을 정하는 곳. release 기본값은 0 이라 그대로 실행하면 응답이 없거나 오류가 납니다 → 직접 값을 채워 실행하며 응답 변화를 비교하세요 (자유롭게 변경 OK).
  • test_*.py — 합격 기준 검증용 값. 변경 금지.

이 문제에서 바꾸는 줄 — record_gen.py 의 아래 줄(ego 추종 컨트롤러). release 기본값은 0 이니 값을 채우세요. leading 운전용 pp_lead 는 시나리오 fixture 라 그대로 둡니다:

5합격 기준 pytest

알고리즘 형태(history 변환·cubic 합성 디테일)는 자유 — 동작 명세(behavioral spec)만 검증합니다.

  1. 인터페이스 — history 가 4점 이상이면 (4, 1) column 반환, 4점 미만이면 zero coeff.
  2. 윈도우 동작 — LeadingTargetTracker 가 max_history 초과 시 가장 오래된 항목 제거.
  3. 폐루프 추종 — leading 이 lane keep, ego 가 그 자취를 추종, 30초 동안 tail 평균 |follow_err| < 0.5 m, peak < 1.5 m, 충돌 없음(min gap > 0).
주의: history 의 회전·시프트 변환이 어긋나면 path 가 빗나가 임계값을 초과합니다.

6힌트

  • 왜 history 를 변환하나 — history 점은 ego local frame 좌표입니다. ego 가 한 스텝 움직이면 frame 자체가 회전·전진하므로, 직전 점들도 새 frame 기준으로 다시 표현해야 합니다 (역회전 후 역시프트).
  • 회전량 은 yaw_rate · dt, 전진량 은 vx · dt 입니다. yaw_rate 가 거의 0(직진)일 때와 곡선일 때의 시프트 식이 다릅니다 — 0 나눗셈에 주의하세요.
  • cubic path — history 점들을 polyfit 한 뒤, ego origin (0, 0) 을 지나고 시작 heading 이 0 이 되도록 제약을 거는 3차 다항식을 만듭니다. 점이 4개 미만이면 안전하게 zero coeff 를 반환하세요.
  • 윈도우는 append 후 길이 초과 시 가장 오래된 항목을 pop(0) 하면 유지됩니다.
